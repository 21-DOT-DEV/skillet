## v0.47 — 2026-07-08

**F16 capture-hardening review round 5 (4 minor concerns; 2 acted, 2 deferred with rationale).** (1) **Hard links — guarded (defense-in-depth).** Grounded's content-read escalates a hard link from benign to a leak exactly as a symlink does, so leaving it open was inconsistent: a produced regular file with **link count > 1** (`st_nlink`, another directory entry — possibly a same-fs host file; cross-fs hard links are impossible) is now disclosed (`FileContent.hardlink: true`, with size) and never read. Regression test: a `link()` to a host secret is disclosed, unread, no leak. The *robust* fix — workspace-level isolation so no reader (grounded, text listing, workspace diff) can be fooled — is deferred to a future sandbox-hardening pass (§9.2 territory), since it's broader than grounded. (2) **`--judge` shadowing rename.** The `judge` `@Option` is renamed `judgeSelection` (flag stays `--judge`) so it no longer visually shadows the `judge`/`judgeCfg` config parameter inside `buildAdapterAndJudge`. (3) **`decodeText` continuation edge — documented, not changed:** an absurd sub-scalar cap could trim leading stray continuation bytes to empty text; unreachable at the 32 KiB cap (a real binary's first 32 KiB carries a NUL or non-continuation invalid byte → binary), and a "fix" adds its own absurd-cap false-positive. (4) **`snapshotStaged` full-fixture read — deferred:** fixtures are maintainer-controlled and copied into the workspace (so `nlink = 1`, not skill-under-test data), making this robustness-against-yourself, not a boundary; bounded reads are a noted follow-up. +1 test. Detail: [Specs/011](Specs/011-grounded-judge/plan.md). 356 tests green. ROADMAP → v1.16.5.

## v0.46 — 2026-07-08

**F16 capture-hardening review round 4 (4 reviewed findings, all valid).** (1) **CORRECTNESS** — `WorkspaceManager.decodeText`'s truncation trim removed *any* high-bit trailing byte, including invalid ones (`0xFF`/`0xFE`), so a binary file whose 32 KiB prefix ended in such a byte (JPEG `0xFF`, UTF-16 BOM `0xFF 0xFE`, …) was stripped and mis-classified as truncated *text* the judge would then grade — violating the "non-UTF-8 withheld as binary" rule. Fixed: trim only a genuine incomplete trailing scalar — up to 3 continuation bytes (`10xxxxxx`) then a real lead byte (`0xC2…0xF4`); an invalid byte or a run of > 3 continuations is left in place, so the strict decode falls through to `nil` ⇒ binary. Regression test: valid UTF-8 then `0xFF` at the cap boundary is now binary. (2) `fileType`'s doc comment claimed "via lstat (does not follow symlinks)" — inaccurate for `attributesOfItem`; reworded to "from its metadata attributes" (moot regardless, since the symlink guards run first). (3) Added a timeout/`ProcessError` catch-path test to match the round-3 "process/timeout failures persist evidence" claim (a `TimeoutWritingAdapter` writes then throws). (4) `--judge grounded-judge` on a trigger-only run now prints a note ("applies to behavioral evals; none are running…") instead of silently ignoring the flag. +3 tests. Detail: [Specs/011](Specs/011-grounded-judge/plan.md). 355 tests green. ROADMAP → v1.16.4.

## v0.43 — 2026-07-08

**F16 capture-hardening review round (5 reviewed findings, all valid; code + tests, no design-decision change).** The grounded judge's produced-file capture (`WorkspaceManager.readProducedContents`) is hardened against special files and lossy transforms: (1) **HIGH** — non-regular files (FIFO / socket / device) are **never opened** — a `FileHandle` read on a pipe blocks forever, so a skill under test could `mkfifo` in the sandbox and hang capture *after* the harness timeout (which only wraps the harness call, not evidence capture); they are disclosed (`special: true`) without a read, and the file-type check runs *before* `unchanged`/`boundedRead` so no special file is ever opened. (2) **MEDIUM** — non-UTF-8 bytes are withheld as `binary` instead of lossy-decoded (`String(decoding:as:)` had replaced invalid bytes with U+FFFD; only NUL was caught): strict UTF-8 validation, with an incomplete trailing scalar trimmed when the read was truncated at the cap (a character split by the cap stays clean text, minus the partial). (3) **MEDIUM** — `FileContent` gains `sizeBytes` so a withheld binary/special file discloses its size, not just "withheld". (4) **LOW** — the deletion pass uses **lstat** existence, so a staged file replaced by a *dangling* symlink is recorded once (modified symlink), not also as deleted. (5) **LOW** — doc date drift corrected. All fixes serve the standing "any omission/cut disclosed, never silently transformed" rule (§9.4 / plan D-3). +4 regression tests, incl. a FIFO DoS test with a `.timeLimit` guard so a reintroduced hang fails rather than hanging CI. Detail: [Specs/011](Specs/011-grounded-judge/plan.md). 346 tests green. ROADMAP → v1.16.1.
