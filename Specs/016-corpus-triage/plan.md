## 1. Outcome

A maintainer points `skillet triage` at a repo with captured recordings and gets, for free and deterministically: a per-skill failure taxonomy (which problem classes recur, how often, how badly), durable finding files the future gates engine can count, live links to the human's friction notes from the same sessions, staleness disclosure, and the next step. Error analysis over observed behavior — never imagined failures (constitution: codify what you discover).

## 2. Scope

**In:** `AnalysisKit` = pure `TriageEngine` (clustering/routing/staleness/join + the rule → lever/confidence table) + effectful `CorpusLoader` (A4); `SarifLog` read accessors in EDDCore (round 1); `TriageCommand` (discovery/selection, evidence I/O via the seam, rendering) **registered in `SkilletCommand.subcommands`**, plus the next-step wiring — `score` and `capture` output gains the `skillet triage` suggestion (P3; round-1 additional); `skillet.triage/1` JSON codec + goldens; RenderKit tty table; tests; docs ripple on landing.

**Out (explicitly):** Track B / `--code-feedback` (Phase 8); F34 contradictions; F35/F36 drift; `--rescore` (roadmap note, D1); finding *refresh* (needs F30's in-place editor — until then stdout always shows fresh counts while files keep first-write counts); `skillet next` itself (Phase 5).

## 3. CLI contract

```
skillet triage [<skill>] [--since <date>] [--dry-run] [--json]
```

- Default: aggregate → cluster → **write missing finding files** (D3) → print taxonomy. `--dry-run`: identical taxonomy, no writes — rows keep their would-be `new: findings/<id>.md` paths and a single `(dry-run: no files written)` footer marks the run (as-built, round 2 — one footer, not a per-row marker).
- Per cluster the table shows: cluster (rule slug) · hits · recordings affected/total · worst level · lever (with caveat) · confidence · file status. The **contractual** status values are the JSON's `file_status`: `new` / `exists` / `closed-still-firing` (frozen, `skillet.triage/1`); the human renderings — `new: findings/<id>.md`, `exists (<state>)`, `closed — still firing; reopen if it's back` — are **illustrative, not an API** (AGENTS: "Human TTY output is *not* an API"; round 3 aligned this wording to the as-built strings).
- Header lines: staleness note (D1) when applicable; coverage note (A7) when applicable.
- Per cluster: `↳ shares session(s) with friction: <ids>` when the computed join (D6) is non-empty.
- Footer (A2, round 2 — the shared registration guard): actionable suggestions name **only registered subcommands** (AGENTS binding rule). While `next` is unregistered: `→ review findings under evaluations/<skill>/findings/ — \`skillet next\` picks them up once it lands (Phase 5)` (capture's exact fallback style for the unshipped `friction add`). Once Phase 5 registers `next`, `LintCommand.nextSteps`'s `registered`-set pattern flips the footer to `→ skillet next` automatically — which is when the roadmap's "output ends by pointing at `skillet next`" metric activates.
- Exit 0 with findings (A3); empty corpus → exit 0, "no recordings yet → skillet capture --skill <s> --slug <x>".

## 4. Data flow (per selected skill — shell steps do I/O, core steps are pure; A4)

**Shell — `CorpusLoader` (AnalysisKit) + the command:**
1. **Enumerate bundles (round 2 — union, not meta-only):** stems = the **union** of `sessions/*.session-meta.json` and `sessions/*.audit-input.sarif`, grouped by stem from one directory listing — enumerating by session-meta alone would make a meta-less bundle invisible, so step 2's promised disclosure could never fire (the silent-skip anti-pattern; batch practice routes every unprocessable input to a *visible* channel rather than dropping it — the dead-letter pattern). Apply `--since` to stems. Traversal is **symlink-guarded** (`ProjectKit.SafeFile.isSymlink` per entry, never followed — the scoring/staging precedent; round-1 additional). Full six-file bundle-layout validation stays `bundle verify`'s job (design §7.2) — triage discloses gaps only in the two files it reads.
2. **Decode per stem** (round 1 — both files, not just the SARIF): `session-meta.json` → `SessionMeta` (its existing `skillVersion`/`model` accessors feed A6's unanimity rule) **and** `*.audit-input.sarif` → `SarifLog` — the **tolerant raw reader** (its doc comment: "exactly the tolerant-reader posture F8 wants"; [Fowler, TolerantReader](https://martinfowler.com/bliki/TolerantReader.html)). `SarifLog` **gains read accessors** for the fields triage needs — per-run `tool.driver.version` and `results[].(ruleId, level, message)` — traversing the raw `JSONValue` (EDDCore; `SarifDocument` stays emit-only). A stem missing either file, or unparseable → a disclosed input row, never a crash (malformed input is data). Yields in-memory `(stem, meta, sarifSlice)` triples.
3. **Decode existing evidence** (seam, command layer): all `findings/*.md` (collect `cluster` fields + states) and `friction/*.md` (ids + sessions) via `EvidenceFrontmatter`, symlink-guarded likewise.

**Core — pure `TriageEngine` (in-memory inputs only; no filesystem):**
4. **Cluster (D2):** group hits by `ruleId` (drop `S000` → coverage note); hits, distinct stems, worst level, first example message. Cluster slug comes from the **§5 fixed table** (`ruleId → slug`) — **no SARIF rules-catalog lookup** (round 1: the table is the id→name map by construction, and old bundles' catalogs stay irrelevant).
5. **Staleness (D1):** recorded `tool.driver.version` ≠ running version → note.
6. **Synthesize (A6) + join (D6):** `Finding` values for clusters with no existing file carrying that `cluster`; cluster sessions ∩ friction sessions → linked friction ids.

**Shell:**
7. **Write missing findings** via `EvidenceFrontmatter.encode` (skip under `--dry-run`); **render** tty table / `skillet.triage/1` JSON.
