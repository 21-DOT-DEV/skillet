## 1. Goal

Ship `skillet run <skill>` — execute a skill's behavioral evals in fresh sandboxes, grade each
`expected_behavior` line with the (text) judge, and report **`pass^k`** reliability. The entry point
(design D2): answers "does this skill work?" in one command, zero workflow required. It fills the
**live half of F6** (`ClaudeCodeAdapter.run()` + §9.2 skill-injection) behind one opt-in, env-gated
paid smoke — every free test runs on the replay seam.

**Success criteria (roadmap F7):**
- `run <skill>` runs every eval *k* times in a fresh `Workspace`, prints a per-eval **PASS/FAIL/FLAKY** table + aggregate `pass^k`, and exits `1` on any failure.
- An eval passes a trial only if *all* criteria pass; a planted **claimed-but-not-created** case **FAILs** (existence/claim-mismatch, via the post-run workspace listing + trace facts). File-*content* correctness — "surface compliance without substance" for file-writing skills — is the grounded judge (F16).
- `pass^k` is computed at **observed k** (= `min` recorded trials across evals) and recomputes identically **offline from the committed `evaluations/benchmark.json`** (per-eval `passed`/`total`), so deleting the `.skillet/` cache loses nothing.
- **No paid surprise (design P9):** `run` estimates the trial count up front and won't spend above `runs.confirm_above_trials` without `--yes`; `--dry-run` previews the plan; `--no-input` fails instead of prompting. Free CI never spends (replay seam).
- Executes a trivial task end-to-end through the **live** `claude-code` adapter (**`probe` version+auth**, `run` + §9.2 injection), returning a parseable `RawTrace` — covered by one opt-in env-gated live smoke (the only paid *test*; free CI uses replay).
- `RunKit`/`JudgeKit` land; `pass^k` math is **pure** (EDDCore); `swift build` + `swift test` green.

- **`EDDCore`** additions: pure `pass^k` aggregation + the `skillet.run/1` report; pure `Verdict`/`TrialResult`/`EvalResult` models. (Run-record codecs already ship in F8.)
- **`HarnessKit`**: implement `ClaudeCodeAdapter.run()` (live) — inject the skill (`SkillSet.only`) into the workspace's discovery path, run `claude -p --output-format stream-json`, return `RawTrace`; the live **`probe()`** (real version/auth — the F6-deferred validation); `parseTrace` already exists. **Extend `ProcessLauncher`** — working directory (cwd) + timeout (+ optional environment); see §4.2.

The other family codecs (`timing`/`metrics`/`eval_metadata`) exist (F8) but full population needs usage parsing (Phase 8); F7 writes benchmark + grading.

## 3. Architecture (targets touched)

```
skillet (executable, .Cxx)   RunCommand: discover skill → load evals → resolve claude → run → render + records
  → ProjectKit          discover skill dir; SkillReader reads SKILL.md + evals.json bytes + gives the skill dir path (adapter injection copies SKILL.md + references/ from that path)
  → ConfigYAML (.Cxx)   loadConfig → SkilletConfig.{judge, runs}
  → RunKit  (effectful) Runner: Workspace lifecycle · evals×k loop (serial) · per-trial timeout · record emit
      → HarnessKit        ClaudeCodeAdapter.run() LIVE (inject .only skill, `claude -p`, parseTrace) · ProcessLauncher(+timeout) · ReplayAdapter
      → JudgeKit          Judge protocol · TextJudge (`claude -p --model`) · ReplayJudge
      → EDDCore           pass^k aggregation (pure) · run-record codecs (F8) · skillet.run/1
  → RenderKit           PASS / FAIL / FLAKY table + pass^k summary + next-step
```

**Layering:** `pass^k` is **pure** in EDDCore (recomputes offline — constitution III). `RunKit`
orchestrates (owns the `Workspace` + loop); the **adapter** privately materializes `.only` (it knows
claude-code's discovery path, §9.2). `JudgeKit` stays **decoupled from `HarnessKit`**: the executable
resolves the `claude` binary once (F6's `BinaryResolver`) and injects the path into **both** the
adapter and the judge, each launching via `swift-subprocess`. The live `run()` + judge are validated
**only** by the env-gated smoke; free CI runs `ReplayAdapter` + `ReplayJudge` end-to-end.

## 4. Detailed design

### 4.1 EDDCore (pure)
- **`Verdict`** — `criterion`, `passed`, `rationale`, `judgeId`, `model`, `judgePromptVersion`. **`TrialResult`** — `exitClass` (`passed`/`failed`/`timeout`) + `verdicts`; a trial **passes iff every verdict passes and the exit class is `passed`**. **`EvalResult`** — an eval's trials.
- **`pass^k` aggregation** (pure fn): **observed k (run-level) = `min` over evals of recorded-trial count** — one common basis per run/harness/arm, so a lost trial can't let one eval claim higher reliability than another (design §4 vocab). Per eval: **PASS** iff *all its* recorded trials pass · **FAIL** iff zero pass · **FLAKY** iff `0 < passes < recorded` — evaluated on the eval's *own* recorded set, **not** truncated to `observed_k`. Aggregate `pass^k` = fraction of evals PASS, **reported at the run's observed k** (at observed k < 2 → **"variance unmeasurable"**, not a number). Takes decoded results, so it re-derives identically from the **committed `evaluations/benchmark.json`** (per-eval `passed`/`total`) — *not* the `.skillet/` cache (constitution P2/D3: cache loses nothing).
- **`RunReport`** (`SchemaIdentified = skillet.run/1`): a run-level **`observed_k`** (= `min` recorded across evals) + aggregate `pass^k` + counts; **per-eval rows expose `passes`/`recorded`** (each eval's *own* recorded count) and a PASS/FAIL/FLAKY status derived from that recorded set — **never** truncated to `observed_k`, so a failed recorded trial is never hidden. (`observed_k` is the run-level interpretation floor for the aggregate, not the per-row denominator.) The `--json` payload.

### 4.2 HarnessKit — the live `run()` (the F6 seam)
- `ClaudeCodeAdapter.run(_ task, in workspace, skills:)`:
  - **Inject `.only(load: [skill])`** — copy the skill **bundle** (`SKILL.md` + `references/` + `scripts/`/`assets/` if present — everything claude-code discovery loads to *run* the skill) into `<workspace>/.claude/skills/<name>/` (the project-level discovery path; confirmed against the real `~/.claude/skills` + project `.claude/skills` layout). **Exclude `evaluations/`** (the test definitions — the model must never see expected answers/grading) and any `.skillet/`/run artifacts. `visible:` siblings → documented seam (no refusal evals in Phase 1).
  - Run `claude -p "<task.query>" --output-format stream-json` (with `--verbose` if required for full events) with **cwd = workspace** via `ProcessLauncher` (the resolved binary) — the **machine-readable streamed JSONL** mode, **not** the default plain-text `-p` (which `parseTrace` cannot consume — it silently skips non-JSON/non-`user|assistant` lines, yielding an empty `Trace`). Capture stdout → `RawTrace(harness: claude-code, raw:)`. `parseTrace` (F6) → `Trace`; the skill firing shows in `Trace.skillInvocations`. **Validation gap:** F6's `parseTrace` was built against the real *native-session* JSONL; if `stream-json`'s line shapes diverge, F7 reconciles `parseTrace` (or falls back to exporting the post-run native session file) — proven at the live smoke (the trace must carry the skill invocation + a `workspaceDiff`). The fake-launcher tests **assert the exact launch args** so an implementer can't silently capture plain text.
- **`ProcessLauncher` — signature change** (today it takes executable + args only): add **`workingDirectory`** (cwd = workspace — required so `claude` discovers the project-level `.claude/skills/`), a **`timeout`** (hard kill on expiry → a `timeout` exit class; `swift-subprocess` cancellation, **no** escalation — F18), and an optional **`environment`** (inherited by default so `claude` reads its own credentials; full per-adapter env-*hygiene* scrubbing is Phase 7 / F51). The fake `ProcessLauncher` records cwd/timeout/env for assertions.
- `ReplayAdapter.run()` already returns a canned `Trace` for CI.

### 4.3 JudgeKit (new, effectful)
- **`Judge`** protocol: `func verdict(for criterion: String, evidence: JudgeEvidence) async throws -> Verdict`. **`JudgeEvidence` carries the response text, the normalized `Trace`, *and* the post-run workspace** — exactly the design's judge-evidence contract (response text + trace + workspace, never response-text-alone, because file-output criteria are where "surface compliance" hides, §9.4). F7's text judge uses the response text **plus a listing of the post-run workspace** (the *actual* files present after the run — `RunKit` lists the sandbox before destroying it; ground truth, unlike the harness-reported `Trace.workspaceDiff` with its F6 gap D) **plus a compact summary of trace-derived facts** (skill invocations, tool calls, files touched — from the parsed `Trace`). So a "claimed to write X, no X exists" case **FAILs**, and "the skill never fired" is checkable. Reading file *contents* from the workspace (the full grounded judge — *created-but-wrong/empty*) is **Phase 2 (F16)** — the same `JudgeEvidence` already carries the workspace **and full `Trace`** for it, so F16 adds no protocol change. **F7's text judge covers the existence/claim-mismatch + trace-fact class** (not file-content quality).
- **`TextJudge`** — builds a deterministic, versioned grading prompt (criterion + the response text + the post-run workspace listing + a compact trace summary — skill invocations / tool calls / files touched → "PASS/FAIL + one-line rationale", phrased so the reply parses unambiguously), shells `claude -p "<prompt>" --model <judge.model>` via the injected launcher/path (the `claude-code` adapter as the judge backend), parses the verdict. Stamps `judgeId` (`text/claude-code`), `model`, `judgePromptVersion` (a constant bumped when the prompt changes). A malformed reply **fails safe** (verdict = fail, raw reply as rationale), never a crash.
- **`ReplayJudge`** — canned verdicts keyed by criterion (CI pipeline tests). Plus an **evidence-asserting** fake judge for the claimed-but-not-created test: it inspects `JudgeEvidence`, **asserts the post-run listing (+ trace) is present**, and decides from it (FAIL when a claimed file isn't in the listing) — so the test proves RunKit actually gathered *and passed* the listing, which a criterion-keyed canned verdict cannot.

### 4.4 RunKit (new, effectful)
- **`Workspace`** — the `HarnessKit.Workspace` type (RunKit drives it via a **workspace manager**; it does *not* redefine the sandbox type — design §11). Lifecycle: `create()` (fresh temp dir), `stage(files:)` (eval `files[]`), **`listing()`** (the actual files present *after* the run — passed to the judge as evidence), `destroy()` (unless `--keep-workspace`). The harness-reported `Trace.workspaceDiff` is also recorded for forensics, but the **judge's existence check uses the ground-truth `listing()`**; the grounded judge (F16) reads file *contents* from the same workspace.
- **`Runner.run(source, config, adapter, judge) -> RunReport`**: for each eval, for k = `runs.k` trials (serial; `concurrency: 1`): create workspace → `adapter.run(task, in: ws, skills: .only(load: [skill]))` under `runs.timeout` → `parseTrace` → **gather `listing()`** → judge each `expectation` (evidence = response text + listing + trace) → `TrialResult` → write the trial record → destroy. Aggregate via EDDCore `pass^k`.
- **Records — committed truth + gitignored cache.** (1) The skill's **committed, frozen `skills/<skill>/evaluations/benchmark.json` + `grading.json`** (design §7.1/§7.2) — the eval-viewer's external-reader contract, via the F8 `BenchmarkFile`/`GradingFile` codecs (byte-compatible). `benchmark.json`'s `runs[]` carry **per-eval `passed`/`total`/`pass_rate`** — **the authoritative source `pass^k` re-derives from** (per-eval `passed`/`total` + observed-k = `min` total → the same `pass^k`), so it survives deleting the cache (constitution P2/D3: everything recomputable from committed `evaluations/`). `grading.json` holds the per-eval expectation verdicts (`{text, passed, evidence}` + summary). This is the **run-record-family producer obligation Spec 006 assigns to F7** (benchmark + grading; `timing`/`metrics`/`eval_metadata` codecs exist but full population awaits usage parsing, Phase 8). (2) The **gitignored cache** `.skillet/runs/<ts>/`: `plan.json` + `trials/<n>/{raw.jsonl, trace.json, verdicts.json, metadata.json}` (exit class, timing; design §7.1 also lists `output.sarif` — **omitted in F7**, scorers are Phase 2/F17) — the **replay source + per-trial forensics** for the current local run, **deletable** (nothing lost: benchmark.json's per-eval counts survive). skillet *writes* both; the human commits the `evaluations/` files (P5).

### 4.5 executable + RenderKit
- **`RunCommand`** (`skillet run [<skill>…] [--runs <k>] [--yes] [--no-input] [-n|--dry-run] [--keep-workspace]`): name-only selection (F4 idiom); `loadConfig` → `runs`/`judge`; **`--runs <k>` overrides `runs.k`** (and feeds the estimate); **decode `evals.json`** (present-but-corrupt → exit `4` artifact error; absent/empty → exit `2` with "no evals to run"); **estimate trials (`evals × k`) and gate spend** — if > `runs.confirm_above_trials` (default 25), confirm on the TTY unless `--yes`; `--no-input` fails (`2`) with the estimate; `-n`/`--dry-run` (the design's global short flag, §5.3) prints the plan + estimate and exits `0` without spending; **`adapter.probe()` before any paid trial** (binary resolved + version not-denylisted + auth) — failure → exit `3` with a remedy. An **auto-discovered banned** binary is *refused* here (exit `3`) — `run` never spends on a known-bad version; the *automatic fallback* to another non-banned binary is **Phase 7 / F50**, not F7. Then build `ClaudeCodeAdapter` + `TextJudge` (or `ReplayAdapter`+`ReplayJudge` under a **hidden, test-only** replay flag — the *public* `--record`/`--replay` is Phase 2 / F19); `Runner.run`; emit records; render. **Exit codes (§5.4):** `0` all PASS · `1` any non-PASS (FAIL *or* FLAKY — `pass^k` demands all k) · `2` bad usage / unknown skill / no evals · `3` harness missing/auth/banned (probe) · `4` corrupt `evals.json` or run artifact.
- **`RenderKit.renderRun(RunReport)`**: a table (EVAL · STATUS · `passes`/`recorded`) + an aggregate line (`pass^k` at the run's `observed_k`) + next-step (suggest `skillet next`/`triage` once they exist, else a generic hint). `--json` → `skillet.run/1`.

### 4.6 Config
- `SkilletConfig.Runs` (`k: Int = 3`, `timeout: String = "10m"`, `concurrency: Int = 1`, `confirmAboveTrials: Int = 25`) + `SkilletConfig.Judge` (`provider: String`, `model: String`), snake_case CodingKeys + `decodeIfPresent` defaults. The F2 template also carries **`infra_retries`**, which stays **unmodeled and ignored until F18** (infra-retry lands there; F7's `Runs` model doesn't read it — unknown keys are ignored on decode), so the template needs no change for it. **A prerequisite config amendment (design §5.2 config block + revision-log, *and* the `init` template): `judge.provider` default `anthropic-api` → `claude-code`** — **applied as F7's *first* step (task 1)**, not deferred to docs-sync, because the default is the *paid provider* and must name one F7 implements (otherwise the shipped default errors). Until task 1 lands, the design/template still say `anthropic-api`. `judge.provider: claude-code` means **use the `claude-code` adapter as the judge backend** (the "any adapter id with the judging capability" reading from §5.2; for Phase 1 that = shelling the *resolved* `claude` CLI via that adapter, not a separate binary). `model` stays `claude-sonnet-4-6` (the `claude` CLI targets it via `--model`).

## 5. Test plan (TDD)

- **HarnessKitTests**: `ClaudeCodeAdapter.run()` injection + launch behind the **fakeable `ProcessLauncher`** (asserts the skill is staged into `<ws>/.claude/skills/<name>/` and the launch args are `-p … --output-format stream-json` with cwd=ws — **not** plain-text `-p`); timeout via the fake.
- **IntegrationTests**: `skillet run <skill>` on a fixture repo via the **replay** path → PASS/FAIL/FLAKY table + exit codes (0 all-pass · 1 any non-pass · 2 unknown skill / no-evals · **3 harness missing/banned via a fake probe · 4 corrupt `evals.json`**) + `--json` `skillet.run/1` + records written (incl. `evaluations/benchmark.json`) + `pass^k` recomputes offline; **spend gate** — `-n`/`--dry-run` previews then exits `0` (nothing spent), `--no-input` above `confirm_above_trials` → exit `2` with the estimate. **DocsTests** claim.
- **Live smoke** (opt-in, env-gated `SKILLET_LIVE_SMOKE=1`, `.tags(.slow)`, skipped in free CI): end-to-end through the **real** `claude` — **`probe` (version + auth + not-denylisted)**, then `run` + §9.2 injection — graded by the **real** `claude`-CLI judge → a parseable trace carrying the **skill invocation** + a verdict. The only paid test (validates the F6-deferred live `probe`/`run`/injection paths).

## 7. Risks & assumptions
- **No live `claude-code`/judge in CI** — the whole pipeline is exercised on `ReplayAdapter` + `ReplayJudge`; the live `run()` + injection + judge are validated only by the one opt-in env-gated smoke (the F6/F7 discipline). The injection path (`<ws>/.claude/skills/<name>/`) is confirmed against the real layout but proven only at the smoke.
- **`parseTrace` ↔ `--output-format stream-json` compatibility** — F6's parser was built against the real *native-session* JSONL; F7's live `run()` captures `-p --output-format stream-json`. If the line shapes diverge, F7 reconciles `parseTrace` (or exports the post-run native session file instead) — validated only at the smoke. The fake-launcher tests pin the launch args so an implementer can't silently capture plain text and emit an empty `Trace`.
- **`pass^k` pure + offline-recomputable** (constitution III + P2/D3): the aggregation re-derives identically from the **committed `evaluations/benchmark.json`** (per-eval `passed`/`total`) — *not* the `.skillet/` cache, which is deletable — locked by a test that aggregates live then re-aggregates from `benchmark.json` with the cache removed.
- **Judge non-determinism** — the text judge calls a model; CI never does (ReplayJudge). The prompt is versioned (`judgePromptVersion`); a malformed reply fails safe.
- **Judge evidence scope** — F7's text judge sees response text + the post-run workspace **listing** (catches *claimed-but-not-done*); *created-but-wrong/empty* needs file-*content* reading = the **grounded judge (F16)**. `JudgeEvidence` already carries the workspace (design §9.4 contract), so F16 adds content reading **without** changing the `Judge` protocol.
- **Timeout only, no escalation/retry** (F7) — one hard per-trial timeout stops a hung `claude` from holing the run; SIGTERM→grace→SIGKILL, infra-retry, and flaky-gating are **F18**. Design **§13 and §10 (the RunKit scheduler text, which today assigns the watchdog + `infra_retries` to RunKit)** are reconciled to match this boundary.
- **Spend is consented, not deferred (design P9; constitution VI SHOULD).** `skillet run` is a user-facing *paid* command (it shells `claude` + the judge), so F7 estimates the trial count up front, confirms above `confirm_above_trials` unless `--yes`, supports `--no-input` (CI) and `--dry-run` (preview) — *no paid surprise* (the Phase-1 metric). **Free CI stays free (constitution V):** every PR test runs the replay seam; the one paid path is the opt-in env-gated smoke. (Real $-cost numbers wait on `Trace.usage` parsing, Phase 8; F7 estimates by trial count.)
- **Secrets/artifacts (VI)** — fixtures + replays are synthetic; no real `~/.claude` logs or API keys committed.

## 8. Definition of done
`skillet run <skill>` runs k trials/eval in fresh workspaces, prints a PASS/FAIL/FLAKY table + aggregate `pass^k` (at observed k = `min` across evals), exits `1` unless all evals PASS (`2` bad usage/no-evals, `3` harness missing/auth/banned via `probe`, `4` corrupt `evals.json`); a planted **claimed-but-not-created** eval **FAILs** (via the post-run workspace listing); **the trial count is estimated up front and spend is gated** (`confirm_above_trials` / `--yes` / `--no-input` / `-n`,`--dry-run`); the shipped `judge.provider` default names an **implemented** provider (`claude-code`, task 1 — a hard gate); the run **writes** the gitignored `.skillet/runs/<ts>/` cache and the skill's committed frozen `evaluations/benchmark.json` + `grading.json` (F8 codecs); `pass^k` recomputes identically offline **from the committed `benchmark.json`** (the cache is deletable — nothing lost); the live `run()`+injection+judge work end-to-end under the env-gated smoke; `RunKit`/`JudgeKit` land, `pass^k` pure in EDDCore; `swift build` + `swift test` green; docs synced (incl. design **§5.2**/§9.4/§10/§13 reconciliation + the `init` template `judge.provider` default + roadmap **Phase 1 → COMPLETE**).
