## 1. Goal & success criteria

Ship the **Discover** step: one command turns the session you just finished into a normalized, scored, greppable evidence bundle — cheaper than skipping it (P4), so the habit sticks. This bundle is the raw material Phase 4 error-analysis reads (northstar gap #1).

**F26 deliverables (roadmap success metrics):**

1. `skillet capture --skill <s> --slug <x>` (newest session by default) writes a **frozen bundle** — `<date>-<slug>.{transcript.md, diff, bodies.json, audit-input.sarif, session-meta.json}` — **plus** the additive `<date>-<slug>.trace.json` (the normalized `skillet.trace/1`), and runs the **deterministic scorers into the bundle's SARIF**.
2. When **corrective turns** are detected, the closing line suggests `skillet friction add` (with the turn count).

**Non-goals (explicit owners):** checkpoint slicing / feedback preservation → **F27**; secret redaction engine → **F32** (014); bundle `inspect`/`verify`/`list`/`stats`/`backfill` → **F28**; the `friction add` command itself → **F30**; the producer-side `.audit-baseline.sarif` path → follow-up with F28's directionality check.

## 3. Architecture

DAG-clean placement (design §11 strict-downward, **corrected R5-3 to match `Package.swift`**): `EDDCore` is the base; its **direct** consumers are `TraceKit`, `ProjectKit`, `RenderKit`, `LintKit`, and `ScoreKit` (`ScoreKit` → `EDDCore` + `ProjectKit`, **not** `TraceKit`). `CorpusKit` (new) → **`EDDCore` + `TraceKit` + `ProjectKit`** — a peer of `HarnessKit` (both are `TraceKit` consumers). So `ScoreKit`, `CorpusKit`, and `HarnessKit` are **peers the `capture` command orchestrates** (as `ScoreCommand` wires `ScoreRunner`) — HarnessKit (locate/export + build the git runner), TraceKit (normalize + detect), ScoreKit (SARIF over staged bodies), CorpusKit (assemble + write). One new kit + `CorpusKitTests`. *(The earlier line wrongly routed `ScoreKit`/`ProjectKit` through `TraceKit`; they depend only on `EDDCore`.)*

**CorpusKit imports no process launcher (R1-2).** `ProcessLauncher` lives in HarnessKit (the sole sanctioned launcher, constitution VI), a CorpusKit sibling — so CorpusKit cannot and does not run `git`. Instead the **command** builds a `GitDiffProvider` closure — **`@Sendable (URL workspace, Set<String> excludedPaths) async throws -> (diff: String, touched: Set<String>)`** (R2-3: `git` already knows the changed-file list, so the provider returns it directly; **no diff-parser to port**, and the list is authoritative + always consistent with the diff. The input **`excludedPaths`** (R3, log-b) are workspace-relative paths to omit from the *untracked-file* scan — e.g. an exported transcript sitting in the workspace — ported from the predecessor's `excludedRelativePaths`) — on HarnessKit's `ProcessLauncher`, and passes the `(diff, touched)` pair into CorpusKit. CorpusKit's deps stay `EDDCore` + `TraceKit` + `ProjectKit`; tests inject a stub provider.

**Artifacts flow as one structured value — `BundleText` (R2-1).** `CorpusKit` defines `struct BundleText: Sendable { var transcript: String; var diff: String; var bodies: [String: String]; var trace: Trace }`. The trace is the **structured `Trace`**, not flattened to text, so (a) secrets in `Trace.turns[].toolCalls[].input` — tool arguments, a documented leak vector — are reachable to the cleaner, and (b) `trace.json` can be re-encoded validly after redaction.

**The `Sanitizer` seam (D-5) is injected, `async` (R1-1), and a *command step* — not a writer parameter (R2-4).** `CorpusKit` defines `protocol Sanitizer: Sendable { func sanitize(_ artifacts: BundleText) async throws -> (redacted: BundleText, report: SanitizationReport) }` (async because the real conformer shells a subprocess) and ships `NoopSanitizer` (returns input unchanged — the **test double only, never a production default**; the constitution forbids writing an un-scrubbed bundle, so the real 014 scanner is a **prerequisite** of the write path and `capture` is exposure-gated on it — R3). The **command** runs the steps in order — assemble `BundleText` → `sanitizer.sanitize(...)` → **score the redacted bodies** (ScoreKit) → `write(...)` — because scoring is a *sibling* module the writer can't call, so redaction must happen *between* assembly and write, orchestrated above both. **Enforced-path guard (R2-1):** `SessionBundleWriter.write(_ redacted: BundleText, sarif: SarifDocument, sanitization: SanitizationReport, meta: SessionMeta, force: Bool) async throws` **requires the `SanitizationReport`** — there is no compilable path that writes a bundle without having sanitized (and the `SarifDocument` slot fixes the missing-SARIF gap). `CaptureCommand` is already `AsyncParsableCommand`. F32's `BetterleaksSanitizer` lands in a **new `SanitizerKit` library** (deps `HarnessKit` + `CorpusKit` — R1-7, a §11 map deviation recorded in §6), injected by the command; **no CorpusKit change** when it lands.

### 3.1 The capture pipeline (design §6.1 mermaid, minus the F32 redact node this slice)

```
locate (adapter, newest)  →  export → RawTrace  →  normalize: parse(jsonl) → Trace   [command orchestrates all steps]
   → render transcript.md (from Trace)                          [D-4, net-new]
   → git provider(session workspace) → (diff, touched)          [R2-3: provider returns the file list too]
   → read touched text-file bodies into memory {relpath: text}  [ported+generalized — R1-8]
   → assemble BundleText{transcript, diff, bodies, trace}       [R2-1: carries the structured Trace]
   → sanitizer.sanitize(BundleText) → (redacted, report)        [D-5 async COMMAND step, BEFORE score — R2-4]
        real scanner (014) redacts + fails closed if it can't run [R3: prerequisite; NoopSanitizer = test double only]
   → stage redacted.bodies to a temp dir; ScoreRunner(tempDir) → audit SARIF   [R1-8: scorers need real files,
        not the JSON wrapper; temp dir skillet-created + defer-cleaned]         [D-2, replaces copy-external]
   → SessionBundleWriter.write(redacted, sarif, report, meta)   [R2-1 enforced-path: report REQUIRED to write]
        → .transcript.md · .diff · .bodies.json · .trace.json · .audit-input.sarif · .session-meta.json (schema 2)
   → detect corrective turns (Trace.turns) → closing nudge      [D-3, registration-filtered — R1-6]
```
Sanitization covers **every** text field of every artifact — transcript, diff, each body, and every `String` in the structured `Trace` (incl. `toolCall.input`) — so a secret echoed in a tool argument or the diff is caught too (R2-1). `.bodies.json` (`{relpath: text}`) is the redacted bodies serialized; scoring runs over the *staged files*, never that wrapper (R1-8).

Collision handling, atomic writes, and `written[]` accounting port from `SessionCapture.capture`. Every artifact shares the `<date>-<slug>` stem; `date` defaults to today (UTC) or `--date`.

### 3.2 Default (newest) session resolution (A2, net-new)
`~/.claude/projects/<enc(target-dir)>/` where `target-dir` (default cwd) is the **captured session's workspace** (R4-3: claude-code keys its store by the dir it ran in, so resolution must use `--target-dir`, not cwd, or it misses the session) and `<enc(...)>` is claude-code's path encoding (`/` → `-`; verified against the live store this repo runs in). Pick the newest `*.jsonl` by mtime, **excluding skillet's own in-flight session**. **Concrete exclusion mechanism (R3, log-a):** (1) if `$CLAUDE_SESSION_ID`/`$CLAUDE_SESSION_FILE` is exported, exclude that file exactly; (2) else treat a candidate as *in-flight* when its mtime is within a short freshness window **and** its last JSONL record is not a completed turn (i.e. the file is still being appended) — skip it; (3) if after exclusion the chosen candidate is still ambiguously fresh, **print the pick and require confirmation** (TTY) or an explicit `--session` (non-TTY) rather than guess. `--session <uuid|path>` always bypasses resolution. Degrades loudly per §9.1 for non-capture harnesses.

### 3.3 Implementation clarifications (R8 — from review, decided)
- **C1 Exposure-gate mechanism (concrete):** `capture` is **not added to `SkilletCommand.subcommands`** until `SanitizerKit` is a linked dependency of the executable **and** `CaptureCommand` constructs+injects a real `BetterleaksSanitizer` (no `NoopSanitizer` in the wired path). So "capture is reachable" ⇔ "the real scanner is compiled in" — enforced by the subcommand registration + the `write(sanitization:)` required arg (R2-1), not by policy.
- **C2 CLI flags:** the **newest session is the implicit default**; `--session <ref>` selects a specific one. D-1's full flag list is `[--session <ref>] --skill <s> --slug <x> [--harness <id>] [--target-dir <path>] [--date <YYYY-MM-DD>] [--force] [--model <m>] [--skill-version <v>] [--fail-on-secret] [--secret-scanner-path <p>]` — **`--date` (default today, UTC)** added to the list. *(Revised 2026-07-11: the `--last` flag + its mutual-exclusion check were removed — see the post-impl log.)*
- **C3 Scanner success guard:** the **authoritative** guard is **"parseable findings JSON ⇒ ran"** (empty `[]` = clean), never the exit code; passing `--exit-code 0` is an optional nicety, not required (some builds may not expose it) — the parser stands on its own.
- **C4 Recursive `Trace` redaction:** use a **`Redactable` protocol** (`func redacting(_ f: (String) -> String) -> Self`), **not `Mirror`** — reflection is read-only (can't write back) and fragile; each type (`Trace`/`Turn`/`ToolCall`/`WorkspaceDiff`/…) explicitly maps its `String` fields through `f` and recurses into children. Explicit, mutating-capable, unit-testable.
- **C5 SARIF path remapping:** stage each redacted body at `<tempDir>/<bundle-relative-path>` (preserving relative structure), so `ScoreRunner`'s emitted `artifactLocation.uri` values come out **bundle-relative by construction**; assert this in a test, and remap if any absolute/temp path leaks (so `audit-input.sarif` stays meaningful after the temp dir is deleted).
- **C6 `GitDiffProvider` untracked files:** `git diff` omits untracked by default — port the predecessor's **tracked + untracked** provider: `git diff --no-renames` **plus** `git ls-files --others --exclude-standard -z` then `git diff --no-index /dev/null <path>` per untracked entry (exit 0/1 both acceptable), honoring the `excludedPaths` set (R3 log-b).

## 4. Reconciliations & the pieces that differ from the port

### 4.1 Generalize skill identity
Drop the hardcoded `knownSkills` set; accept any skill discoverable under `skills_root` (ProjectKit). Slug regex `^[a-z0-9][a-z0-9-]*$` and `YYYY-MM-DD` date shape port verbatim.

### 4.2 Transcript is rendered, not copied (D-4, net-new)
The predecessor copied a Zed-exported markdown file. skillet has no exported transcript — it renders one from `Trace` (or directly from the JSONL turns): `## User` / `## Assistant` headers, `**Tool Call: <name>**` blocks, thinking elided or fenced. Faithful + readable for open coding; **not** a byte-match of Zed's idiom. `TranscriptFeedback`'s header grammar (`## User` / `## Assistant`) is honored so the ported detector *could* run on the text — but D-3 runs it on `Trace.turns` instead (§4.4).

### 4.3 SARIF is scorer-generated, not copied (D-2, reconciliation)
The predecessor **copied** `<target>/.docc-suite-state/audit-baseline.sarif` and renamed by role. skillet is harness/skill-neutral and cannot assume that file exists, so it **runs ScoreKit's deterministic scorers (F17) over the captured produced bodies** → `<slug>.audit-input.sarif`. This is the intended skillet evolution (roadmap metric: "runs the deterministic scorers into the bundle's SARIF"), and it composes with F32: **redact bodies → score → emit**, so the SARIF is clean by construction. Producer skills (which *emit* their own SARIF) → `.audit-baseline.sarif` is a follow-up; capture stamps the role now.

### 4.4 Corrective detection: port the algorithm, move the input (D-3)
Port `TranscriptFeedback`'s **recall-first** rule — index user turns, drop turn 0 (the opening task, not a correction) and bare continuations (`continue`/`go`/`go ahead`/`proceed`), keep every other non-empty user turn "for the human to validate" — but source turns from **`Trace.turns` (role == user)** rather than parsing `## User` headers, satisfying design §9.3 (harness-independent). Rationale it supersedes the earlier phrase-list recommendation: the downstream consumer is the Track B **judge** (Phase 4), so a high-recall upstream filter is correct — a precision phrase-list would drop real corrections before the judge codes them. The count feeds the closing nudge, which is **registration-filtered** (R1-6, the `ScoreCommand.nextSteps` idiom): capture always prints the observation (`trace: N turns, M corrective`), and appends the runnable `skillet friction add --skill <s> --sessions <id>` line **only when `friction` is a registered subcommand** — so F26 never suggests a command that errors, and the line auto-activates when F30 registers `friction`. (Until then the nudge names the intent + "arrives with F30" rather than a broken command.)

### 4.5 `session-meta.json` v2 (D-7)
Port the 7-field snake_case codec verbatim (keys sorted, pretty-printed — matches the live corpus), add optional `sanitization: {scanner, version?, redactions?}` (absent in v1 corpus; `nil` on decode; **no `disabled`** — R5, capture never writes unsanitized). The full field set is declared here (R2-6) so F32 adds behavior, never a field, to this frozen boundary. `harness` is a **plain `String`** (R2-5): the codec lives in `EDDCore`, *below* `TraceKit`, so it cannot import `HarnessID` — the command stringifies `adapter.id.rawValue` (predecessor's `SessionMeta.harness` is already `String`).

**Resolution (R5-2 — grounded in existing types; the earlier A5 over-claimed):** `harness` ← adapter id; `model` ← **the session JSONL's `model` field** (verified present — e.g. `claude-opus-4-8`) ?? `--model` ?? `"unknown"`; `skill_version` ← `--skill-version` ?? `"unknown"` — **the frontmatter source is dropped** (`SkillFrontmatter` has only `name`/`description`, *no* `version`), and the predecessor's SARIF-version source is dropped too (skillet's `audit-input.sarif` carries *skillet's* version, not the skill's); `captured_at` ← session end (or now); `harness_version` is **not** in meta (rides `.trace.json`). *(Reading the model from the JSONL is a small net-new add to the export path — the log carries it; a `metadata.version` frontmatter source for `skill_version` is a possible future enhancement.)* `"unknown"` sentinel preserved (counted separately, never poisons diversity — §8).

## 5. Test plan (TDD) — free-first, offline
- **Golden codec** (`EDDCoreTests`): `SessionMeta` is a **`RawJSONObject`** with typed accessors (R6-4 — like `SarifLog`/`RunRecord`; a plain `Codable` struct would **drop** unknown keys, violating the frozen round-trip policy). v1 (no `sanitization`) and v2 round-trip byte-stable against **synthetic** fixtures mirroring the real corpus shape (constitution VI); the `sanitization` object round-trips its **three fields `{scanner, version?, redactions?}`** (R5 — `disabled` removed); `harness` decodes as `String` (R2-5); **unknown-key round-trip** (a hand-added key survives re-encode); `"unknown"` sentinel.
- **CorpusKit**: bundle layout (all six stems), collision refuse + `--force` clobber, `diffWasEmpty` warn, generalized bodies extraction (touched text files), SARIF role-rename, `NoopSanitizer` stamps `scanner:none`; **`BundleText` round-trips the structured `Trace`** (redaction of a planted secret in `toolCall.input` re-encodes valid `trace.json` — R2-1); **`write` cannot be called without a `SanitizationReport`** (enforced-path, a compile-time guard exercised by the tests); the injected `GitDiffProvider` stub returns `(diff, touched)` and drives bodies (R2-3, no diff-parser).
- **TraceKit**: corrective detector over synthetic `Trace`s — drops opening + continuations, keeps substantive; empty/one-turn traces ⇒ 0.
