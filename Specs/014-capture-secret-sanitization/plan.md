## Research (industry grounding for the assumptions)
- **`betterleaks` is real & current** — MIT, pure-Go, offline BPE detection (98.6% recall), a drop-in Gitleaks replacement ([BleepingComputer](https://www.bleepingcomputer.com/news/security/betterleaks-a-new-open-source-secrets-scanner-to-replace-gitleaks/), [Help Net Security](https://www.helpnetsecurity.com/2026/03/19/betterleaks-open-source-secrets-scanner/)).
- **Scanners detect; they don't rewrite files** — Gitleaks `--redact` only scrubs *its own report output*; "it does not modify repository data beyond reporting findings" ([gitleaks](https://github.com/gitleaks/gitleaks), [Harness ref](https://developer.harness.io/docs/security-testing-orchestration/sto-techref-category/gitleaks-scanner-reference/)). ⇒ **D-3** (skillet redacts).
- **Value/pattern redaction is the norm for unstructured bodies** (OTel redaction processor: value-based "regardless of where they appear" for log bodies) ([OTel redaction](https://oneuptime.com/blog/post/2026-02-06-redact-sensitive-data-pii-opentelemetry-pipeline/view)). ⇒ **D-4** (also solves the cross-artifact echo).
- **Derived artifacts leak independently** — "build logs, artifacts … can preserve secrets after the original file is fixed" ([coverage](https://nhimg.org/faq/how-do-i-implement-secrets-scanning-in-a-cicd-pipeline/)). ⇒ **D-5** (scan all text artifacts).
- **CI convention = block/fail on confirmed secret**, but since our written bundle is *already redacted*, the non-zero is a **rotation alarm** ("assume the credential escaped — act immediately"), not a write-guard ([CI gate](https://stevekinney.com/courses/self-testing-ai-agents/secret-scanning-with-gitleaks)). ⇒ **A3** (write-then-exit-1).

## 1. Goal & success criteria
Stop captured evidence from leaking credentials into the committed corpus. Capture scans the transcript/diff/bodies/trace and **redacts secrets in place** before writing; the raw secret never enters the repo (it remains only in the harness's native store, which capture reads but does not commit).

**F32 deliverables (roadmap success metrics):**
1. A planted AWS key / GitHub token / private key in a session is redacted to a typed marker (`[REDACTED:aws-access-key]`) in the committed bundle; the raw value appears **nowhere** in the repo.
2. Default **redacts + prints a review report without blocking** (P4); `--fail-on-secret` exits **1** for CI; if the scanner **can't run**, capture **fails closed** (never writes an unsanitized bundle).
3. A false positive is silenced with a one-line allowlist / path exemption; **redaction provenance** (scanner, version, count) is recorded in `session-meta.json`.

## 2. Scope

- A new **`SanitizerKit` library** (+ **`SanitizerKitTests`**, R2-b) housing a `SecretScanner` seam + **`BetterleaksSanitizer`** conforming to F26's `async` `Sanitizer` over `BundleText`: resolve `betterleaks` (D-2), run it detection-only/offline, parse its findings, and perform **span-merge value-based in-place redaction** (D-3/D-4/R2-2) across **all text artifacts + every `Trace` string field incl. `toolCall.input`** (D-5/R2-1) — the command runs it **before** the ScoreKit step.
- **Fail model** (D-6): `can't-run` → fail-closed (§4); `found-secrets` → redact + report + exit 0 (P4); `--fail-on-secret` → write-then-exit-**1** (A3). **No `--no-sanitize` bypass** (R5 — always sanitize; false positives via `exempt_paths`).
- **Provenance**: fill the frozen `session-meta.sanitization` — canonical shape **`{scanner: String, version?: String, redactions?: Int}`** (R5, matching F26 §2/§4.5; `version`/`redactions` optional; `redactions` is a **single Int count**; **no `disabled` field** — capture always sanitizes). Any **per-rule / per-artifact breakdown lives only in the in-memory `SanitizationReport`** used for the printed report — it is **not** serialized into the frozen boundary.
- **Exemptions**: `betterleaks`'s own allowlist (`.betterleaks.toml`) honored, **plus** a skillet `sanitize.exempt_paths` config (glob, whole-segment). Every redaction appears in the report so an exemption is a one-line fix.
- **`doctor`**: a `capture.secret-scanner` row — betterleaks resolves? version? — surfaced as a **warning** free-of-charge (so absence is known before capture), escalating to the fail-closed refusal only at capture time.
- **RenderKit**: the redaction report (rule, artifact, count — never the secret). **Tests** (§5). **Docs/Package** (§6).

### Out of scope (→ owner)

## 3. Architecture
`BetterleaksSanitizer` lives in a **new `SanitizerKit` library** — deps `HarnessKit` (the sanctioned `ProcessLauncher`) + `CorpusKit` (the `async` `Sanitizer` protocol) + `EDDCore` — a top-tier module beside RunKit, so its JSON parser + cross-artifact redactor are **`@testable` in isolation** (R1-7). The `capture` command **injects** it as the `Sanitizer` it runs on the `BundleText` **before scoring** (R2-4), then passes the redacted artifacts + `SanitizationReport` into `SessionBundleWriter.write` (which *requires* the report — the enforced-path guard). No CorpusKit or `Sanitizer`-protocol change; F32 is "provide a non-no-op conformer *in its own library* + wire it in `capture`." This is a §11 module-map deviation (the map implied the executable hosts the sanitizer) — recorded in §6. Resolution reuses `BinaryResolver` (flag `--secret-scanner-path` > `SKILLET_BETTERLEAKS_BIN` > `[sanitize].scanner_path` > `PATH`), matching harness-binary discipline (§9.1) — printable winning link, no auto-probing private caches. **Errors (R6-3, hybrid — the codebase's own pattern):** `SanitizerKit` throws a **domain `SanitizerError`** (`.scannerNotFound`, `.scannerFailed`) — like `HarnessError`/`ProcessError`; the `capture` command **translates** it at the boundary into a user-facing **`EDDError.sanitizerNotFound`** (exit 3, remedy → `--secret-scanner-path` / `SKILLET_BETTERLEAKS_BIN` / `sanitize.scanner_path` — **not** `harnessNotFound`'s wrong `--harness-path`), or the fail-closed can't-run error.

### 3.1 The redaction algorithm (D-3/D-4; conforms to F26's `Sanitizer` over `BundleText`)
1. Gather **all** scannable text from the `BundleText` (R2-1): `transcript`, `diff`, every `bodies` value, **and every `String` field of the structured `Trace`** — critically `turns[].text` **and `turns[].toolCalls[].input`** (tool arguments — a documented secret-leak vector). The `Trace` is structured, not flattened, so it re-encodes to valid `trace.json` after redaction.
2. Run `betterleaks` **per artifact, via stdin** — `betterleaks stdin -f json -r - --set-attr path=<artifact-key>` (**R6-1, spike-verified against betterleaks 1.6.1**): piping keeps the raw text **in memory, never on disk** (the security win — best practice over temp files), and `--set-attr path=` populates each finding's `File`, so findings carry a filename label *without* temp files. The text is piped through `ProcessLauncher` using swift-subprocess's **`input: .data(...)`** (**spike-verified, 0.2.1** — a small additive `input:` param on the launcher; the isolated spike compiled + ran end-to-end). Decode with PascalCase `CodingKeys` (`ruleID="RuleID"`, `secret="Secret"`, `match="Match"`, `file="File"` — R4-1); **success = a parseable JSON array** (`[]` = clean → exit 0; findings → exit 1) — decided by *parsing*, **never the exit code** (R3-3); unparseable/absent output ⇒ can't-run ⇒ fail closed. The scanner's JSON *contains raw secrets* → held **in-memory, never written to disk**.
3. Collect findings `(Secret, RuleID, File)`; **drop any whose `File` ∈ `sanitize.exempt_paths`** (**R6-2**, a post-scan false-positive filter the sanitizer owns — feasible precisely because `--set-attr` labels each finding). The surviving **`Secret` values** (with `RuleID`) are the redaction set.
4. **Span-merge redaction (R2-2, the Presidio "resolve-then-redact" pattern — value-based):** for each text field, compute **every occurrence span** of every detected secret value, **merge overlapping and adjacent spans** into maximal non-overlapping spans, then replace each merged span with `[REDACTED:<rule>]` in one pass (a span covering >1 rule → `[REDACTED:multiple]`). Order-independent, deterministic, and provably fragment-free — a shorter secret that is a substring of a longer one can never leave a dangling tail. Applied uniformly across all artifacts **and** all `Trace` string fields (catches echoes betterleaks flagged in only one copy).
5. Re-encode the redacted `Trace` structurally, return the redacted `BundleText` + the in-memory `SanitizationReport` (which **may** carry per-rule/per-artifact detail for the printed report). Only its **summary** — `{scanner, version?, redactions?: Int}` (R5) — is serialized to the frozen `session-meta.sanitization`. The command then scores the *redacted, staged* bodies (F26 §3.1, R1-8) and calls `write` — so the emitted SARIF never contains a raw secret.

### 4. Fail model (D-6) — the crux of the safety posture
| Situation | Behavior | Exit |
|---|---|---|
| Scanner **can't run** (unresolved binary, crash/signal, **unparseable / no findings JSON**) | **Fail closed** — no bundle written; typed error + remedy (`betterleaks not found — install … or set SKILLET_BETTERLEAKS_BIN=…; capture will not write an unsanitized bundle`); predicted by the `doctor` row | 3 |
| Scanner runs, **finds secrets** | Redact in place, **write** bundle, print redaction report (P4/P5) | 0 |
| `--fail-on-secret` + secrets found | Redact + **write** the safe bundle, then fail (rotation alarm; report preserved — A3) | **1** (`measuredFailure`, design §6.1 — R1-5) |
| Scanner runs, **clean** | Write; report "0 redactions"; `sanitization:{scanner,version,redactions:0}` | 0 |

*(There is deliberately **no "write unsanitized" row** — R5. Capture always sanitizes; a false positive is silenced surgically via `sanitize.exempt_paths` / the `betterleaks` allowlist, never a blanket disable.)*

**Exit-code discipline (R3-3, issue 3).** betterleaks/gitleaks reuse exit **`1`** for *both* "leaks found" **and** "error" (and can even exit `0` on stderr noise) — so a naive "non-zero ⇒ can't run" rule would fail-closed on **every** finding. skillet instead runs the scanner with **findings-as-success** (`--exit-code 0` / equivalent) and decides success by **whether it returned parseable findings JSON**, not by the exit code: parseable ⇒ *ran* (findings = the array, possibly empty, taking the redact-and-write path); unparseable/absent ⇒ *can't run* ⇒ fail closed (exit 3). A normal "found secrets" run is thus never misclassified as a scanner failure.

## 5. Test plan (TDD) — offline, no real betterleaks in CI
- **`SecretScanner` seam** via a **`StubScanner`** returning planted findings: span-merge redaction replaces **all** occurrences across artifacts + **every `Trace` string field including `toolCall.input`** (R2-1), typed markers correct, secret value absent from every written byte, and the redacted `Trace` re-encodes to valid `trace.json`.
- **Overlap (R2-2)**: planted secrets `abc` **and** `abcd` (containment) and two mid-overlapping secrets ⇒ **no dangling fragment** in output, and the result is **byte-identical** regardless of finding order (determinism).
- **Ordering**: redact-before-score — a planted secret in a body never appears in the emitted `audit-input.sarif`.
- **Fail-closed**: a scanner that throws / exits non-zero / emits garbage ⇒ **no files written**, exit 3, remedy text.
- **Flags**: `--fail-on-secret` writes + **exit 1**; `sanitize.exempt_paths` skips a false-positive path; **no `--no-sanitize`** exists (R5 — a "write unsanitized" invocation must be impossible to construct).
- **Parser (R4-1)**: a golden betterleaks JSON fixture with **PascalCase** `RuleID`/`Secret`/`Match` (matching the real binary) decodes correctly; redaction uses `Secret` (not `Match`); the scanner's raw-secret JSON never reaches disk.
- **Provenance**: `session-meta.sanitization` fields (golden, extends the F26 v2 meta codec test).
- **`doctor`**: betterleaks-missing ⇒ warning row (not a hard fail outside capture).
- **Report**: never prints the secret; prints rule + counts.
- *(Optional, opt-in, non-CI)* a real-`betterleaks` smoke behind an env guard (like the F14 live smoke) — never gates.
